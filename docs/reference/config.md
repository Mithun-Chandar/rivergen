# rivergen.config.json Reference

RiverGen works without any configuration file. All defaults match the standard
v2 monorepo layout (`apps/api`, `apps/web`, `packages/shared`). Create
`rivergen.config.json` only when your project diverges from those paths or when
you want to set `dbImport`, `sharedPackage`, or `auditDir`.

---

## File location and loading

Place `rivergen.config.json` at the project root (next to `package.json`).

RiverGen performs a **shallow deep-merge**: the top-level objects `api`, `web`,
and `packages` are each merged with their defaults independently. You can
override a single field inside `api` without restating every other `api` field.
Scalar top-level fields (`dbImport`, `sharedPackage`, `auditDir`,
`applyRecordsDir`) replace their defaults entirely when present.

If the file is missing, all defaults apply. If the file contains invalid JSON,
RiverGen logs a warning and falls back to defaults.

---

## When you need a config file

| Situation                                                          | Action                                           |
| ------------------------------------------------------------------ | ------------------------------------------------ |
| Standard monorepo (`apps/api`, `apps/web`, `packages/shared`)      | No config file needed                            |
| API app is at a different path (e.g., `server/`)                   | Override `api.srcRoot` and related `api.*` paths |
| Web app is at a different path (e.g., `frontend/`)                 | Override `web.srcRoot` and related `web.*` paths |
| Different shared package name                                      | Set `sharedPackage`                              |
| Consistent DB client import path across all domains                | Set `dbImport`                                   |
| Audit artifact files live outside the default `witness/` directory | Set `auditDir`                                   |

---

## Three key fields

These three fields are the most commonly customized. They are all optional and
have no effect on path resolution — they control generated code content and
gate behavior.

### `dbImport`

```json
"dbImport": "{ prisma } from \"../lib/db\""
```

The import specifier for your database client. When set, every generated
`mutations.ts` file opens with:

```typescript
import { prisma } from "../lib/db";
```

When omitted, the generated file contains a TODO comment instead:

```typescript
// TODO: import your DB client here
```

Set this once if your DB client is accessible at a consistent relative path
from every domain's mutations file. Leave it unset if paths differ per domain
or if you prefer to fill in the import manually.

### `sharedPackage`

```json
"sharedPackage": "@acme/shared"
```

Default: _(none)_

The npm package name that exports `ENTITY_PROJECTIONS`. RiverGen places
generated entity-projection slices under `packages.entityProjectionsDir`, and
the web app's dispatcher barrel imports from this package name. Set this to
your actual shared package name (e.g. `"@acme/shared"`). When omitted, the
generated barrel uses the package name as written by `rivergen init` — check
your `packages.entityProjectionsDir` barrel to confirm the import path.

### `auditDir`

```json
"auditDir": "audit"
```

Default: `"witness"`

The directory that contains the three optional payload audit artifact files:

| File                                 | Purpose                                                |
| ------------------------------------ | ------------------------------------------------------ |
| `phase4-payload-continuity-audit.ts` | `REQUIRED_FIELDS` entries (payload continuity)         |
| `phase5-test-payloads.ts`            | Test payload `publish()` calls (trace coverage replay) |
| `phase6-retained-slice-audit.ts`     | Retained-slice lifecycle and signal assertions         |

**Gate #11 (Event Audit Coverage)** checks that every broadcast event appears in
all three files. If none of the three files exist in `auditDir`, Gate #11 skips
silently — it assumes the project has not yet adopted the payload audit
workflow.

`auditDir` is independent of `web.witnessDir`. Witness files (per-domain field
continuity proofs) are placed under `web.witnessDir`; the audit artifacts above
are manually maintained in a separate location and are not generated by
RiverGen.

## Complete field reference

Fields marked **[deprecated]** are legacy v1 upsert targets. They are kept for
backward compatibility and do not need to be set in new projects. The v2
slice-based generation no longer writes to these paths.

### `api.*`

| Field                   | Default                                                   | Notes                                        |
| ----------------------- | --------------------------------------------------------- | -------------------------------------------- |
| `api.srcRoot`           | `apps/api/src`                                            | Root source directory of the API app         |
| `api.listenersDir`      | `apps/api/src/lib/event-bus-listeners`                    | EventBus listener slice directory            |
| `api.schemasDir`        | `apps/api/src/lib/event-factory/schemas`                  | EventFactory payload schema slice directory  |
| `api.schemasBarrelFile` | `apps/api/src/lib/event-factory/schemas/_index.ts`        | Schema barrel file                           |
| `api.schemasBaseFile`   | `apps/api/src/lib/event-factory/schemas/_base.ts`         | Base Zod schemas (EventEnvelopeSchema etc.)  |
| `api.eventFactoryFile`  | `apps/api/src/lib/event-factory/event-factory.service.ts` | EventFactory service (written by `gen:init`) |
| `api.eventBusFile`      | `apps/api/src/lib/event-bus.service.ts`                   | EventBus service (written by `gen:init`)     |
| `api.packageJsonPath`   | `apps/api/package.json`                                   | Used for dependency enforcement              |
| `api.schemasFile`       | `apps/api/src/lib/event-factory/schemas.ts`               | **[deprecated]** v1 upsert target            |

### `web.*`

| Field                       | Default                                               | Notes                                             |
| --------------------------- | ----------------------------------------------------- | ------------------------------------------------- |
| `web.srcRoot`               | `apps/web/src`                                        | Root source directory of the web app              |
| `web.dispatchersDir`        | `apps/web/src/lib/cache/domain-dispatchers`           | Domain dispatcher slice directory                 |
| `web.dispatchersBarrelFile` | `apps/web/src/lib/cache/domain-dispatchers/_index.ts` | Dispatcher barrel file                            |
| `web.wsBindingsDir`         | `apps/web/src/providers/ws-bindings`                  | WebSocket binding slice directory                 |
| `web.wsBindingsBarrelFile`  | `apps/web/src/providers/ws-bindings/_index.ts`        | WS bindings barrel file                           |
| `web.queryKeysDir`          | `apps/web/src/lib/query-keys`                         | Query key factory slice directory                 |
| `web.queryKeysBarrelFile`   | `apps/web/src/lib/query-keys/_index.ts`               | Query keys barrel file                            |
| `web.entityCacheFile`       | `apps/web/src/lib/cache/entity-cache.ts`              | Entity cache (written by `gen:init`)              |
| `web.stateCacheFile`        | `apps/web/src/lib/cache/state-cache.ts`               | State cache lookup table (written by `gen:init`)  |
| `web.projectionsDir`        | `apps/web/src/lib/projections`                        | Domain projection file directory                  |
| `web.hooksDir`              | `apps/web/src/hooks`                                  | Domain hook file directory                        |
| `web.witnessDir`            | `apps/web/src/witness`                                | Per-domain Witness files (field continuity audit) |
| `web.packageJsonPath`       | `apps/web/package.json`                               | Used for dependency enforcement                   |
| `web.providerFile`          | `apps/web/src/providers/WebSocketProvider.tsx`        | **[deprecated]** v1 upsert target                 |
| `web.dispatcherFile`        | `apps/web/src/lib/cache/state-cache.ts`               | **[deprecated]** v1 upsert target                 |
| `web.queryKeysFile`         | `apps/web/src/lib/query-keys.ts`                      | **[deprecated]** v1 upsert target                 |

### `packages.*`

| Field                                  | Default                                            | Notes                                    |
| -------------------------------------- | -------------------------------------------------- | ---------------------------------------- |
| `packages.entityProjectionsDir`        | `packages/shared/src/entity-projections`           | Entity projection slice directory        |
| `packages.entityProjectionsBarrelFile` | `packages/shared/src/entity-projections/_index.ts` | Entity projections barrel file           |
| `packages.entityProjectionsTypesFile`  | `packages/shared/src/entity-projections/_types.ts` | `EntityProjectionEntry` type definitions |
| `packages.typesEventsFile`             | `packages/types/src/realtime-events.ts`            | **[deprecated]** v1 upsert target        |
| `packages.sharedEntityRegistryFile`    | `packages/shared/src/entity-projections.ts`        | **[deprecated]** v1 upsert target        |

### Top-level fields

| Field             | Default                       | Notes                                                           |
| ----------------- | ----------------------------- | --------------------------------------------------------------- |
| `applyRecordsDir` | `artifacts/gen-apply-records` | Directory where apply records are written for reversibility     |
| `dbImport`        | _(none)_                      | Import specifier for the DB client in generated mutations files |
| `sharedPackage`   | _(none)_                      | Package name that exports `ENTITY_PROJECTIONS`                  |
| `auditDir`        | `witness`                     | Directory containing optional phase4/5/6 audit artifact files   |

## Examples

### Minimal config — only customize what differs

This is the most common pattern. Override the paths that differ from the
defaults and leave everything else out.

```json
{
  "dbImport": "{ prisma } from \"../lib/db\""
}
```

```json
{
  "sharedPackage": "@acme/shared",
  "dbImport": "{ db } from \"../database\""
}
```

### Non-standard directory names

If the API app lives at `server/` and the web app at `client/`:

```json
{
  "api": {
    "srcRoot": "server/src",
    "listenersDir": "server/src/lib/event-bus-listeners",
    "schemasDir": "server/src/lib/event-factory/schemas",
    "schemasBarrelFile": "server/src/lib/event-factory/schemas/_index.ts",
    "schemasBaseFile": "server/src/lib/event-factory/schemas/_base.ts",
    "eventFactoryFile": "server/src/lib/event-factory/event-factory.service.ts",
    "eventBusFile": "server/src/lib/event-bus.service.ts",
    "packageJsonPath": "server/package.json"
  },
  "web": {
    "srcRoot": "client/src",
    "dispatchersDir": "client/src/lib/cache/domain-dispatchers",
    "dispatchersBarrelFile": "client/src/lib/cache/domain-dispatchers/_index.ts",
    "wsBindingsDir": "client/src/providers/ws-bindings",
    "wsBindingsBarrelFile": "client/src/providers/ws-bindings/_index.ts",
    "queryKeysDir": "client/src/lib/query-keys",
    "queryKeysBarrelFile": "client/src/lib/query-keys/_index.ts",
    "entityCacheFile": "client/src/lib/cache/entity-cache.ts",
    "stateCacheFile": "client/src/lib/cache/state-cache.ts",
    "projectionsDir": "client/src/lib/projections",
    "hooksDir": "client/src/hooks",
    "witnessDir": "client/src/witness",
    "packageJsonPath": "client/package.json"
  },
  "dbImport": "{ prisma } from \"../lib/db\""
}
```

### Audit artifacts in a non-default location

If your payload audit files live in a top-level `audit/` directory instead of
the default `witness/`:

```json
{
  "auditDir": "audit"
}
```

Gate #11 will then look for `audit/phase4-payload-continuity-audit.ts`,
`audit/phase5-test-payloads.ts`, and `audit/phase6-retained-slice-audit.ts`.
If none of those files exist, the gate skips silently regardless of which
directory is configured.
